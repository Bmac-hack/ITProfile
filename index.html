Project idea 


Overview 

My project would be about an “EnCon” dynamic module that connects to a smartphone app wirelessly / cell network  in order to automate everyday power conservation. Each module would cost around $59-99 depending on how broad it’s function was. Envisage this “module / unit” being able to connect to your car, to your household appliances / lights / switchboard and powering things off that haven’t been used for a set duration of time in order to save electricity.Instead of having several different apps for different energy providers, apps that save fuel, and even communications apps - It would be a central “energy” hub incorporating everything. It would also serve to track consumption on the app so you can see what your overall “Expenditure” is and provide tips on how to improve it.
So for a smart meter, it would add some “Extra smart” to it by plugging into the circuit board with newly designed circuits that are far more robust and communicate to the app wirelessly on your network with a small master chip. This would give people a better indication of peak vs off-peak times and you’d be able to focus on saving energy - and this could also be used for governing bodies as a tracking mechanism for disputes.
I would call this ecosystem “EnCon”  and it would be able to connect to just about everything desired.

Motivation
My motivation for this is based on a personal ethos - why use resources when we don’t need to? By being more conscious and having the right modern tools that can do this - it is absolutely possible.
The broader application of such a product would be useful in areas where resources are starved and very limited. It would have installations on Marine, military, and domestic use. Marine for better power monitoring and management (Eg cruise ship) - Military/police, to record a solid subset of data for investigative usage. This creates a ginormous demand for an extremely wide user base. This is the next generation of eco-conscious people as well as the elderly. Hospital applications

Description

The primary basis of the service would be a web/app based subscription model where the user needs to spend $5 a month for the most basic level of service - the monitoring central hub side of “EnCon” which would focus on far cheaper modules that sell for $29 that do not have any automation capabilities. This is so all energy consumptions / vehicle / communications / appliances / hubs can be monitored in a central location. In this case, “Basic EnCom smartmodules” would be installed to a persons smartmeter / fuse box, connect up various appliances for more detailed costs not just an overall “Feel” of things - and then even somehow connect to the water meter at the property to measure this. All utilities would be connected.
The more capable modules would cost $99 for a basic start-up kit including a few monitoring modules and an “automation” module. The premium product would be $299 and include 6x monitoring modules, and cover a small house with automation (6x lights, 8x power socket modules, smartmeter module, water module) and people who spend $10 p/m on the app would have full access to the really cool automation side of it. This would be the no holds barred version. Car “ECU” module would be available for $49 - there would be several other niche models available. A “Strava” EnCom module would be available for cyclists who commute to work for $35 that works from an ANT+ sensor and collects data from your garmin / Wahoo unit to track how much fuel / consumption has been saved. When driving it would calculate your cost of fuel rather than relying on per 100K Ratings that never really show the cost until you fill up.

A user would simply connect these modules as it if was a network switch or device plugged into their home network and connect every unit up to a Master module included in the box. The master module would need a very high level of security as who wants their house “Hacked” ?
Some kind of “locator” module could be used to refine the more conservative modes to follow the person. This could be a wearable or the smartphone app. So if you have the wearable on you, or your phone - it won’t turn anything off in that room as you’re in it (if it’s a light or something else you could be using)
But if it’s detected you are leaving the house, it will try to gauge this and shut things down as required.

 

It would be a 2 step setup process

Step 1 - power on and enter password when the LED is turned on
Step 2 - it detects all the modules installed and powered on and connects them - It then loads the default “Eco” profile set to turn anything off automatically after 15 minutes.Naturally you wouldnt want your fridge connected to this right…..this is where the monitoring modules would be useful. You’d plug this module into things that didn’t need automation but just monitoring.

Tools and software / Skills required
First I would need a team of engineers to come up with a beta/draft design that can go out to the field followed by an application that would be able to run the “smart” side of the modules to inform the user of all the consumption involved and tips on how to save etc. then, You’d need a factory that can print PCB’s to your specification. Software like AutoDesk Eagle would be able to assist in the design of the PCB using CAD software on some moderately specced PC’s. Kicad could also be used and is free. AutoDesk requires a subscription nowadays when it never used to.Progrmmers would be needed to write the smart phone app and firmware / kernel / software for the smart module onto the onboard memory chips that contain the function of each “EnCon” Module but it would also need some very clever “Hackers” or ITSec people (possibly both) in order to ensure it’s as secure as possible around the clock so people don’t have to worry about things being compromised. They would need to “Harden” the firmware / software as much as possibl and only release updates that were absolutely necessary. There would be cloud servers involved to store customer information and manage subscriptions. Would certainly not encourage “ads” or “selling information” - it would be 100% Ethical and the data on the servers would not be accessible even to staff without law intervening noting that even this would be encrypted data.
